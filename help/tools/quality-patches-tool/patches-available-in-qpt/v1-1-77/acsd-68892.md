# ACSD-68892: Inkonsekvent [!DNL Fastly]-cachelagring för cachelagrade sidor

Korrigeringen ACSD-68892 åtgärdar ett problem där cachebara sidor inte lagras eller hanteras korrekt från cacheminnet [!DNL Fastly], vilket resulterar i inkonsekvent cachningsbeteende och minskad prestanda. Den här korrigeringen är tillgänglig när [[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) 1.1.77 är installerad. Korrigerings-ID är ACSD-68892. Observera att problemet är planerat att åtgärdas i Adobe Commerce 2.4.9.

## Berörda produkter och versioner

**Korrigeringen har skapats för Adobe Commerce-version:**

* Adobe Commerce (alla distributionsmetoder) 2.4.8-p2

**Kompatibel med Adobe Commerce-versioner:**

* Adobe Commerce (alla distributionsmetoder) 2.4.8 - 2.4.8-p3

>[!NOTE]
>
>Korrigeringen kan bli tillämplig för andra versioner med nya [!DNL Quality Patches Tool]-versioner. Om du vill kontrollera om korrigeringen är kompatibel med din Adobe Commerce-version uppdaterar du `magento/quality-patches`-paketet till den senaste versionen och kontrollerar kompatibiliteten på [[!DNL Quality Patches Tool]: Sök efter korrigeringsfiler &#x200B;](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html). Använd patch-ID:t som söknyckelord för att hitta patchen.

## Problem

Cacheable-sidor lagras eller hanteras inte korrekt från cacheminnet [!DNL Fastly], vilket resulterar i inkonsekvent cachningsbeteende och försämrade prestanda.

<u>Steg som ska återskapas</u>:

1. Aktivera [!DNL Fastly]-cachen.
1. Skapa en ny webbplats-/butiksvy.
1. Lägg till den sekundära webbplatskoden i `index.php`:

```
$_SERVER['MAGE_RUN_TYPE'] = 'website';
$_SERVER['MAGE_RUN_CODE'] = 'web2';
```

1. Kontrollera cachekontrollhuvudet med hjälp av vändning:

```
curl -Is https://localhost/m247/pub/ | grep cache-control
```

<u>Förväntade resultat</u>:

Cacheable-sidor cachas i [!DNL Fastly].

<u>Faktiska resultat</u>:

Cacheable-sidor cachelagras inte i [!DNL Fastly], de `max-age=0` som returneras i huvudet.

```
curl -Is https://localhost/m24develop/pub/ |grep cache-control
cache-control: max-age=0, must-revalidate, no-cache, no-store
```

Använd följande länkar beroende på distributionsmetod för att tillämpa enskilda korrigeringsfiler:
